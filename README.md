# Standard Evolver

让 AI 先理解业务，再用数据持续进化业务判断标准的通用 Skill。

Standard Evolver 不是一个复杂系统，也不是一个可视化图谱工具。它是一套轻量工作协议：先让用户说清楚自己的业务，再帮助用户建立一套业务分析标准，之后每次拿到新数据，都用数据去校准这套标准，并把新的判断规则沉淀下来。

一句话概括：

```text
先建立业务标准，再让每一次数据分析成为下一次标准进化的证据。
```

## 它解决什么问题

很多 AI 分析的问题不是模型不会分析，而是缺少稳定的业务标准。

常见情况包括：

- 数据一来就直接分析，但没有先定义“什么算好、什么算差”。
- 同一批数据在不同对话里会被解释成不同结论。
- 用户的业务经验只停留在脑子里，没有沉淀成可复用标准。
- 分析结果看起来有道理，但不知道依据是什么。
- 每次复盘都像重新开始，上一轮经验没有影响下一轮判断。
- 图谱、知识库或项目文档有很多内容，但缺少让数据持续校准标准的流程。

Standard Evolver 的目标是把这些隐性的业务判断显性化，让 AI 的分析不只是“给结论”，而是逐步形成一套可复用、可解释、可迭代的业务标准。

## 核心流程

```text
业务说明 -> 建立标准 -> 分析数据 -> 调整标准 -> 沉淀记忆
```

这也是业务图谱背后的底层逻辑：图谱只是其中一种展示方式，真正重要的是业务标准会随着数据不断变清楚。

## 功能介绍

### 1. 业务理解

Standard Evolver 会先引导用户说明业务，而不是直接进入数据分析。

它会关注：

- 你做什么产品或服务
- 你的用户或客户是谁
- 你希望 AI 帮你判断什么
- 什么结果算好，什么结果算差
- 当前有哪些业务目标、约束和专有词汇
- 哪些判断过去经常出错或摇摆

这一步的价值是让 AI 先进入业务语境，避免用通用逻辑粗暴套用到具体业务上。

### 2. 建立业务标准

在理解业务后，Standard Evolver 会整理第一版业务分析标准。

一套标准通常包括：

- 分析维度：应该从哪些角度看数据
- 分类标签：数据应该被分成哪些类型
- 证据规则：什么证据强，什么证据弱
- 优先级或评分逻辑：哪些信号更重要
- 排除项：哪些情况不能误判
- 输出要求：分析结果应该怎么表达

这一步的作用是让 AI 不再凭感觉分析，而是先有一套可以复用的判断框架。

### 3. 按标准分析数据

当用户提供数据、案例、素材、聊天记录、项目笔记或客户反馈后，AI 会按当前业务标准去分析。

分析重点包括：

- 这批数据说明了什么
- 哪些证据支持当前判断
- 哪些地方仍然不确定
- 哪些案例挑战了原来的标准
- 哪些信息需要继续补充
- 本次分析是否暴露了标准本身的问题

重点不是只输出结论，而是把“结论从哪里来”说清楚。

### 4. 根据数据进化标准

如果新数据证明原来的标准太粗、太偏或漏了关键情况，就调整标准。

可调整的内容包括：

- 新增一个分析维度
- 修改一个标签定义
- 调整一个证据强弱规则
- 增加一个排除条件
- 记录一个典型案例
- 标记一个暂时不能确定的问题

这里有一个重要原则：每次只改必要的一小部分。不要因为一批数据就重写整套标准，否则无法判断到底是哪条规则产生了效果。

### 5. 沉淀成可复用记忆

最后把真正有价值的内容沉淀下来。

沉淀内容包括：

- 被确认的业务规则
- 被修改的判断标准
- 可复用的案例
- 典型误判
- 仍然未知的问题
- 下一次分析应该注意什么

这样下一轮分析就不是从零开始，而是从更新后的业务标准开始。

## 三种使用模式

### 初始化模式

用于第一次使用时，根据业务描述建立第一版标准。

适合这样问：

```text
用 standard-evolver 帮我先梳理业务，并建立第一版业务分析标准。

我的业务是：
用户/客户是：
我希望 AI 帮我判断：
什么结果算好：
什么结果算差：
```

### 分析模式

用于已经有一套标准后，按标准分析新数据。

适合这样问：

```text
用 standard-evolver 按当前业务标准分析下面这批数据。

当前标准是：
新数据是：
我希望得到的输出是：
```

### 进化模式

用于分析后，根据数据证据更新标准。

适合这样问：

```text
用 standard-evolver 判断这批数据是否需要更新当前标准。

如果需要，只修改被数据证明需要修改的部分，并说明依据。
```

## 推荐输入格式

你不需要一开始就准备得很完整，但越清楚越好。

推荐这样提供信息：

```text
业务背景：
目标用户：
我想让 AI 判断的问题：
什么结果算好：
什么结果算差：
已有标准：
这次要分析的数据：
希望输出形式：
```

如果你还没有已有标准，可以留空，让 Standard Evolver 先帮你建立第一版。

## 输出内容

一次完整输出通常包含五部分：

```text
Business Context：业务背景
Business Standard：当前业务标准
Data Analysis：基于标准的数据分析
Standard Update：标准更新记录
Memory Deposit：可复用记忆沉淀
```

一个简化输出可能长这样：

```text
业务背景：
你正在分析用户对某类产品的真实需求。

当前标准：
把“喜欢”视为弱偏好信号，把“搜索、购买、询价、投诉、招聘、复购”视为更强需求信号。

数据分析：
这批数据里有大量“喜欢”和“想要”，但缺少付费、询价或替代方案搜索，因此不能直接判断为强需求。

标准更新：
新增规则：表达喜欢不能单独作为付费意愿证据，必须结合行动信号判断。

记忆沉淀：
下次分析需求时，先区分偏好信号和需求信号。
```

## 使用示例

假设你在做产品机会分析。

用户提供了一批评论，里面很多人说“这个东西不错”“我喜欢这个功能”。普通 AI 可能直接总结为“用户需求强烈”。但 Standard Evolver 会先建立业务标准：偏好信号和付费信号要分开。

然后它会这样分析：

- “喜欢”是弱信号，只说明兴趣或情绪倾向。
- “搜索替代方案”“询价”“购买”“投诉现有方案”“招聘相关岗位”是更强信号。
- 如果只有喜欢，没有行动证据，就不能直接判断为强需求。

最后它会沉淀一条新规则：

```text
在产品机会分析中，不要把喜欢直接等同于付费意愿。必须检查是否存在行动型需求信号。
```

这就是标准进化：不是多写一段总结，而是让下一次判断更稳。

## 它的优点

### 更稳定

先有标准再分析，减少同一类数据被反复解释成不同结论。

### 更可解释

每个判断都能追溯到业务标准和数据证据，而不是只给一个看似合理的结论。

### 可沉淀

每次分析都会留下可复用的规则、案例和边界，下一次可以直接继承。

### 小步进化

每次只调整一小部分标准，避免因为一次数据波动就推翻整套逻辑。

### 不绑定工具

可以用于 Codex Skill、普通提示词、项目文档、知识库、图谱系统或任何 AI 工作流。

### 适合业务图谱

如果你正在做图谱，Standard Evolver 可以提供图谱背后的判断逻辑：哪些节点有价值，哪些关系只是共现，哪些关系才有强证据。

## 使用建议

- 先从一个具体业务目标开始，不要一开始就覆盖所有场景。
- 第一版标准可以粗糙，但必须能指导下一次分析。
- 每次只改一条或少量规则，方便验证效果。
- 把典型案例沉淀下来，案例比抽象原则更容易复用。
- 区分事实、解释和猜测，不要把临时判断写成确定规则。
- 数据不足时标记“不确定”，不要强行进化标准。
- 不要一开始就做复杂数据库、看板或自动化流程，先让标准循环跑通。
- 如果要公开分享案例，记得去掉隐私、客户名和敏感业务信息。

## 不适合什么

Standard Evolver 不适合用来：

- 直接替代行业专家判断
- 把所有原始资料原封不动存起来
- 一次性生成完整商业策略
- 在没有业务目标的情况下泛泛总结资料
- 没有验证就把猜测固化成标准

它更适合做“业务标准的持续校准层”。
